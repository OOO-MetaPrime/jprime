### Функции

Если часть данных лежит физически в другой БД и нет возможности достать их одним sql-запросом вместе с остальными
данными,
то можно воспользоваться Функциями, которые не зависят от расположения данных и могут обращаться к другим сервисам для
их получения.

#### Принцип работы

``JPDataFunctionCommonService`` проверяет резалт-сет на наличие в значениях полей шаблонов, вызывающих доступные функции
и, если находит,
то вызывает их с переданными в шаблоне аргументами. Функция возвращает данные в виде мапы,
где ключ - составной ключ из шаблона функции и значений всех аргументов,
а значение - результат функции. После этого ``JPDataFunctionCommonService`` ещё раз проходит по резалт-сету
и подменяет шаблоны на результат функции.

Для вызова функции необходимо включить в sql-запрос константу вида:

```
'${<code>.<template>$<arg1>$<arg2>$...$<argN>}'
```

где `<code>` - это код функции,

`<template>` - один из шаблонов функции,

а `<arg1>-<argN>` - аргументы, передаваемые в функцию (порядок важен!). Могут быть переданы по ссылке (на поле из этого
же запроса)
или по значению (в запросе не должно быть полей совпадающих по имени с этим значением).

Пример:

* передача аргумента по ссылке:

```
select worker_id, '${staffTitle.position$worker_id}' as position from ...
```

в функцию `staffTitle`, у которой есть шаблон `position`, будет передан в качестве аргумента значение поля `worker_id`,
после чего в поле `position` будет положен результат работы функции.

* передача аргумента по значению:

```
select worker_id, '${staffTitle.position$123}' as position from ...
```

в функцию `staffTitle`, у которой есть шаблон `position`, будет передано в качестве аргумента `123`, после чего в поле
`position` будет положен результат работы функции.

#### Типовые функции

##### Генерация штрихкодов

| Код     | Описание                    |
|---------|-----------------------------|
| barcode | Функция генерации штрихкода |

* Шаблоны для вызова функции:

| Шаблон       | Описание                                    |
|--------------|---------------------------------------------|
| ean13Barcode | Шаблон для генерации штрихкода              |
| 2DBarcode    | Шаблон для генерации 2D штрихкода (QR-кода) |

* Аргументы:

| № | Код    | Описание |
|---|--------|----------|
| 1 | text   | Текст    |
| 2 | height | Высота   |
| 3 | width  | Ширина   |
| 4 | margin | Отступ   |

* Пример:

```
select ..., 
    '123456789012' message_1,
    '123456' message_2,    
    '${barcode.ean13Barcode$message_1$200$400$0}' barcode_1,
    '${barcode.2DBarcode$message_2$300$300}' barcode_2
from ...
```

##### Перевод даты в кириллицу

| Код         | Описание                                         |
|-------------|--------------------------------------------------|
| dateToWords | Функция преобразование даты в текстовое описание |

* Шаблоны для вызова функции:

| Шаблон   | Описание                                 |
|----------|------------------------------------------|
| words_1  | первое января две тысячи двадцатого года |
| format_1 | 01 января 2020 г.                        |
| format_2 | 01 января 2020 года                      |
| format_3 | 1 января 2020 г.                         |
| format_4 | 1 января 2020 года                       |
| format_5 | "01" января 2020 г.                      |
| format_6 | "01" января 2020 года                    |
| format_7 | "1" января 2020 г.                       |
| format_8 | "1" января 2020 года                     |

* Аргументы:

| № | Код  | Описание     |
|---|------|--------------|
| 1 | date | Поле с датой |

* Пример:

```
select ..., 
    '2040-10-01' date_field,    
    '${dateToWords.words_1$date_field}' value_1,
    '${dateToWords.format_1$date_field}' value_2
from ...
```
