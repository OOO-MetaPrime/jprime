# Основной модуль планировщика

Логика планировщика, выполнения задач по расписанию.
Поддерживаются как статический планировщик, так и динамический с возможностью отключения задач и запуска новых

## Настройки расписания

Периодичность запуска настраивается согласно типового cron-подхода

### Структура формата

```
  ┌───────────── second (0-59)
  │ ┌───────────── minute (0 - 59)
  │ │ ┌───────────── hour (0 - 23)
  │ │ │ ┌───────────── day of the month (1 - 31)
  │ │ │ │ ┌───────────── month (1 - 12) (or JAN-DEC)
  │ │ │ │ │ ┌───────────── day of the week (0 - 7)
  │ │ │ │ │ │          (or MON-SUN -- 0 or 7 is Sunday)
  │ │ │ │ │ │
  * * * * * *
```


| Блок        | Диапазон значений         |
|-------------|---------------------------|
| Секунды     | 0-59                      |
| Минуты      | 0-59                      |
| Часы        | 0-23                      |
| День месяца | 1-31                      |
| Месяц       | 1-12 (или JAN-DEC)        |
| День недели | 0-7 (0 и 7 — воскресенье) |

### Примеры выражений


| Описание                                          | Шаблон                  |
|---------------------------------------------------|-------------------------|
| Каждую секунду                                    | * * * * * *             |
| Каждые 10 секунд                                  | */10 * * * * *          |
| Каждый час в 15 минут                             | 0 15 * * * *            |
| Ежедневно в 1:01 ночи                             | 0 1 1 * * *             |
| Каждый день в 8, 9, 10 утра                       | 0 0 8-10 * * *          |
| По будням (пн-пт) в 10:15                         | 0 15 10 * * 1-5         |
| Каждый месяц первого числа в полночь              | 0 0 0 1 * *             |
| Каждые 15 минут в рабочее время (с 9:00 до 17:00) | 0 */15 9-17 * * *<br /> |

Специальное значение выражения cron, указывающее на отключенный триггер: "-".
Это в первую очередь предназначено для использования с заполнителями ${...}, позволяющими отключать соответствующие запланированные методы извне.

## Использование API для статического запуска задач

Рекомендуется вместо статического запуска через `@Scheduled` использовать вызов `JpScheduleService`

Было:

```java
class TestClass {
  
  @Scheduled(cron = "${jprime.resources.lockmanager.memory.clearTimeout:0 */30 * * * *}")
  public void deleting() {

  }
}
```

Стало:

```java
class TestClass implements JPApplicationStartListener {

  @Value("${jprime.resources.lockmanager.memory.clearTimeout:0 */30 * * * *}")
  private String cron;

  private JpScheduleService scheduleService;

  @Autowired
  private void setScheduleService(JpScheduleService scheduleService) {
    this.scheduleService = scheduleService;
  }

  @Override
  public void applicationStart() {
    scheduleService.schedule(
        "Очистка кеша заблокированных ресурсов",
        JpScheduleTaskCatalog.SYSTEM,
        JpScheduleCron.of(cron),
        this::deleting
    );
  }

  private void deleting() {

  }
}
```
